这份策划文档（PRD）旨在为 Cursor 提供一个清晰的、可执行的框架。既然你要制作一个网页版的 3D Demo，我们将技术栈锁定在 **Three.js** 或 **Babylon.js**（结合 React/Vue 或纯 JS），这是目前网页端实现高画质 3D 效果的主流选择。

# 游戏局内 3D 场景 Demo 策划文档 (PRD)

## 1. 项目概述

* **项目名称：** Dark Dungeon Web Demo
* **风格：** 中世纪暗黑地牢（参考《Dark and Darker》或《艾尔登法环》）
* **核心目标：** 实现基础的 3C（Character, Controls, Camera）体验、UI 交互逻辑及 Debug 系统的网页端 Demo。

---

## 2. 视觉与美术需求

* **环境风格：** 阴暗、潮湿的地牢。石砖墙壁、火炬（动态光源）、祭坛、宝箱。
* **角色模型：** 中世纪骑士（带有简单的行走、攻击、跳跃、下蹲动画）。
* **UI 布局：** 遵循移动端操作逻辑，左侧虚拟摇杆，右侧功能按键群。

---

## 3. 核心功能详细描述

### 3.1 3C 系统 (Character, Controls, Camera)

* **Character (角色):** * 状态流：待机 (Idle) -> 移动 (Run) -> 跳跃 (Jump) -> 下蹲 (Crouch) -> 攻击 (Attack)。
* **Controls (操作):**
* **左侧摇杆：** 屏幕左下角 2D 叠加层，控制角色在 3D 空间内的平面位移。
* **右侧功能键：** 跳跃（按下触发一次向上冲力）、下蹲（按住或切换，降低碰撞体高度）。


* **Camera (摄像机):**
* **第三人称追随：** 摄像机始终锁定在角色后上方。
* **视角旋转：** 触摸/滑动右侧空白区域，计算 $\Delta x$ 和 $\Delta y$ 来旋转摄像机，角色朝向随旋转逻辑调整。



### 3.2 战斗与技能系统

* **攻击键：** 点击触发近战挥砍动作，并伴随“事件飘字”（如：-50, Critical!）。
* **技能按键 (3个)：** * **Skill A:** 瞬间释放。
* **Skill B:** 持续引导/长按。
* **Skill C:** 指向性技能（带指示器）。



### 3.3 UI 与辅助系统

* **小地图 (Minimap):** * 渲染一个正交摄像机的顶视图或基于配置文件的 2D 地图层。
* 实时显示玩家图标，并根据玩家位置同步坐标（$x, z$ 轴）。


* **事件飘字 (Floating Combat Text):** * 在 3D 空间中角色头顶生成 HTML 标签或 Canvas 纹理，向上漂浮并淡出。

---

## 4. Debug 菜单功能 (核心开发需求)

为了方便测试，需在界面一角设置 **Debug 按钮**，点击弹出菜单，可实时修改技能的交互模式：

| 功能项 | 选项 | 说明 |
| --- | --- | --- |
| **技能触发方式** | 点击 (Instant) | 按下即释放 |
|  | 长按 (Charge) | 按住蓄力，松开释放 |
|  | 拖拽摇杆 (Directional) | 拖拽出摇杆控制施法方向 |
| **指示器类型** | 无 / 扇形 / 直线 | 切换类 MOBA 的技能范围显示 |
| **性能信息** | FPS / DrawCall | 监控网页渲染性能 |

---

## 5. 技术实现建议（供 Cursor 参考）

* **渲染引擎：** 使用 `Three.js` 处理场景与模型。
* **物理引擎：** `Cannon.js` 或 `Rapier.js` 处理角色碰撞、地面检测。
* **UI 层：** 采用 CSS/HTML 叠加在 Canvas 之上，方便快速迭代 UI 布局。
* **资源：** 初始阶段建议使用低多边形 (Low-poly) 模型或基础几何体代指。
