# 受击反馈技术规范 (Cursor 存档版)

本文档为受击反馈模块的**技术规范与实现约束**，后续开发请严格参照，避免破坏已实现行为。

---

## 1. 血条反馈模块 (Health Bar)

### 1.1 结构：三层 DOM（不可删减）

血条必须保持**三层 DOM** 结构，顺序与层级固定：

| 层级 | 用途 | 说明 |
|------|------|------|
| **背景层** | 容器底色 | 如 `#hp-container` 背景或 `.world-bar-back`，z-index 最低 |
| **幽灵层 (Ghost)** | 扣血延迟动画 | 显示「扣除闪色」，在主血量收缩后缓慢跟至新值 |
| **主血量层 (Main)** | 当前血量 | 受击瞬间步进，无过渡 |

- **面板血条 (HUD)**：`#hp-container` 内为 `#hp-ghost`（z:1）、`#hp-main`（z:2）；容器本身作背景（z:0）。
- **头顶血条 (World Bar)**：`.world-bar-wrap` 内为 `.world-bar-back`（z:0）、`.world-bar-ghost`（z:1）、`.world-bar-fill`（z:2）。若需受击抖动，应在**内层包裹**（如 `#playerWorldBarShakeWrap`）上施加 `shake-active`，避免覆盖外层定位用 `transform: translate(-50%, -100%)`。

### 1.2 动画规则

- **主血量层**：受击瞬间 **100% 步进**，禁止使用 `transition`（`transition: none !important`）。
- **幽灵层**：使用 **TWEEN.js** 从当前宽度插值到新血量百分比；**时长**由调试参数 `dropSpeed`（秒）决定，即 `duration = dropSpeed * 1000` ms；建议缓动 `Easing.Cubic.Out`。幽灵层颜色使用调试参数 `dropColorFlash`（扣除闪色）。
- **容器层**：受击瞬间在容器（及头顶血条内层）上添加 class **`shake-active`**；**强度**由调试参数 `shakeIntensity`（单位 px）通过 CSS 变量 `--s-int` 传入。动画结束后（约 200ms）移除 `shake-active`。

### 1.3 抖动动画 (globalShake)

- 使用统一 keyframes **`globalShake`**，基于 `--s-int`：
  - `0%` → `translate(0,0)`
  - `25%` → `translate(calc(var(--s-int)*-1), var(--s-int))`
  - `50%` → `translate(var(--s-int), calc(var(--s-int)*-1))`
  - `75%` → `translate(calc(var(--s-int)*-1), calc(var(--s-int)*-1))`
  - `100%` → `translate(0,0)`
- `.shake-active`：`animation: globalShake 0.15s ease-in-out !important;`，并设置 `--s-int` 默认/来自 params。

### 1.4 数值方位 (Value Position)

血条数值（当前/最大 HP 文本）方位支持：

- **Top / 上**
- **Bottom / 下**
- **Left / 左**
- **Right / 右**
- **Inner / 内**：数值在血条**内部居中**显示

实现方式：通过 class 控制容器布局（如 `hp-left`、`hp-right`、`hp-top`、`hp-inner` 等），Inner 为血条内居中，其余为 flex-direction/定位偏移在血条外一侧。

### 1.5 闪白 (Flash)

受击时对**主血量层**（如 `#hp-main`、`#playerWorldBarFill`）做短暂高亮：`filter: brightness(3)`，约 100ms 后恢复 `brightness(1)`。

## 2. HUD 反馈模块 (HUD Text)

### 2.1 跳字逻辑

- **Normal（普通）**：基础字号与样式，**淡出上漂**（如 animType `floatUp`）。
- **Critical（暴击）**：**1.5 倍大小**（如 fontSize 27 vs 18）、**zoomPop 动效**，颜色使用调试参数 **`criticalColor`**；普通颜色使用 **`normalColor`**。

### 2.2 状态弹出

受击时可在 **floatingTexts 容器**（如 `#floatingDamageContainer`）中额外生成**状态 Icon**（如眩晕、减速等），与伤害数字同容器、同套投影与生命周期管理。

### 2.3 暴击图标

暴击跳字前可带暴击图标；图标颜色与跳字颜色一致（或使用 `criticalColor`），若图片加载失败可回退为 SVG/绘制。

## 3. 方位提示 (Directional UI) —— 待实现

### 3.1 功能描述

在**玩家脚下**显示 **2D 箭头**，指向**攻击者**方向，用于提示受击来源。

### 3.2 方向角计算公式

在世界 XZ 平面（Y 为高度）下，箭头指向攻击者的角度：

$$\text{Angle} = \text{atan2}(\text{Attacker.x} - \text{Player.x},\ \text{Attacker.z} - \text{Player.z})$$

- 使用 `Math.atan2(dx, dz)`，其中 `dx = Attacker.x - Player.x`，`dz = Attacker.z - Player.z`。
- 箭头 UI 需根据该角度旋转，与项目坐标系一致（如 Three.js 中 Y 轴向上、相机朝向等）。

### 3.3 反馈表现

- 箭头随**受击事件**弹出显示。
- **0.5 秒**内渐隐（如 opacity 1 → 0）。

## 4. 调试参数 (UE 工具箱) 对照

| 参数名 | 含义 | 影响 |
|--------|------|------|
| `dropSpeed` | 幽灵层下降速度（秒） | TWEEN 时长 = dropSpeed × 1000 ms |
| `shakeIntensity` | 抖动强度 (px) | 写入 `--s-int`，供 globalShake 使用 |
| `dropColorFlash` | 扣除闪色 | 幽灵层背景色 |
| `healthBarValuePosition` | 血条数值位置 | 左/右/上/内 |
| `normalColor` / `criticalColor` | 普通/暴击跳字颜色 | HUD 跳字与图标 |

## 5. 关键实现要点（防破坏检查清单）

- [ ] 血条 DOM 始终三层：背景 → 幽灵 → 主血量；z-index 与 `transition: none` 在主血量层保持。
- [ ] 主血量仅做即时 `width` 更新；幽灵层仅通过 TWEEN 更新 width，且主循环中调用 `TWEEN.update(performance.now())`。
- [ ] 受击时同时为**面板血条容器**与**头顶血条内层**添加 `shake-active`，并设置 `--s-int`；约 200ms 后移除 class。
- [ ] 跳字：Normal 使用 floatUp + normalColor；Critical 使用 zoomPop + 1.5 倍字号 + criticalColor，且可带暴击图标。
- [ ] 方位箭头为待实现功能，公式与表现见第 3 节。
